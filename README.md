# 🏏 IPL Multi-Season SQL Analytics Case Study (2018 - 2019)

Welcome to my SQL Portfolio project! This repository contains an end-to-end relational database analysis comparing the **2018 and 2019 Indian Premier League (IPL) seasons**. 

The goal of this project is to bypass superficial surface-level sports statistics and leverage advanced SQL querying to uncover deep structural patterns in team strategies, multi-season player consistency, and defensive bowling efficiencies.

---

## 🛠️ Advanced SQL & Relational Engineering Toolkit Demonstrated

This project showcases production-level SQL scripting capabilities, moving beyond basic `SELECT` syntax into rigorous analytical engineering:

*   **Multi-Temporal Data Merging (`UNION ALL`):** Pipeline construction across disconnected historical tables to track multi-year athlete progressions natively.
*   **Relational Data Coupling (`INNER JOIN`):** Merging distinct metrics on unique primary key keys (`Player`) to extract cross-discipline data such as true defensive all-rounders.
*   **Advanced String Manipulations (`STRING_AGG`):** Clean dynamic string concatenation to trace and report roster mobility for athletes moving across different franchises.
*   **Analytical Aggregations & Deduplication:** Heavy reliance on conditional filter patterns (`WHERE`), group constraints (`GROUP BY`), value isolation (`DISTINCT`), and sorting frameworks (`ORDER BY`).

---

## 🎯 Strategic Case Study Insights

### 1. Multi-Season Volume Dominance
By joining multi-season records with an iterative combination of `UNION ALL`, `SUM()`, and `GROUP BY`, the analysis maps long-term batting resilience. **KL Rahul** emerged as the undisputed standard of consistency, accumulating a monumental **1,252 runs** over the combined two-year stretch.

### 2. Boundary Density vs Single-Run Reliance
Using custom mathematical operator expressions (`Fours + Sixes`) to compute aggregate boundaries, the dataset exposes modern T20 paradigms. Young talents like **Rishabh Pant** showed dominant boundary dependency, rewriting traditional tactical playbooks through pure strike aggregation.

### 3. Precision Middle-Over Efficiency
Applying strict conditional logic checks (`WHERE Economy_Rate < 7.0`), the system isolated primary tactical anchors. Spin wizards like **Rashid Khan** and technical pacers like **Lungi Ngidi** consistently proved their capability to depress scoring rates and maintain elite efficiency levels under pressure.
